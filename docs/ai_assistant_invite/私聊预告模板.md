# 私聊预告模板 · 拉群前用

> **时机**：拉群前 30 分钟 - 1 天
> **对象**：你想拉进内测群的每个核心员工
> **目的**：让员工心理上有准备，不至于一进群就懵

---

## 模板 A · 标准版（飞书私聊文字）

```
@同事姓名

⏰ 30 分钟后我会拉你进一个"AI 小助手内测群"，先跟你说一声～

▸ 是什么：公司内部新上线的 AI 工作助手（飞书机器人），帮你
  写邮件 / 文案 / 总结文档 / 整理思路 / 拆任务

▸ 为什么拉你：你日常工作场景丰富，反馈对产品打磨很有价值

▸ 占多少时间：每天空闲时随便聊几句，不强制 KPI

▸ 测试时长：5/18 - 5/24（一周）

▸ 退出方式：随时跟我说就行，群里看到不喜欢可以直接退

进群第一条消息会是详细说明，先有个心理准备～
```

---

## 模板 B · 给老板（杨国民）专用

```
杨总，

下午想拉一个"AI 小助手内测群"试运行，先跟您打个招呼。

▸ 测什么：公司内部新上线的 AI 助手在多人群聊场景下的表现
▸ 拉谁：5-10 个核心员工（各部门骨干）+ 您
▸ 期望：跑一周，下周一给您一份反馈数据报告
▸ 风险控制：我每天盯监控，出问题立即下线

您只需要：
  1. 加入群（被动接受）
  2. 偶尔 @ 一下试试（看您方便）
  3. 周日给我几句评价（好 / 差 / 想加啥功能）

不需要您专门处理什么 —— 我会全程兜底。
```

## 模板 C · 给部门负责人专用

```
@张总（部门负责人）

下午要拉一个 "AI 小助手内测群"，邀请您和您部门 1-2 位骨干员工进来试用。

希望您：
  1. 配合内测（每天试几句）
  2. 推荐 1-2 个您部门最适合的员工一起进群
  3. 周日给我反馈"对您部门有用 / 没用"

测试时长一周，不耽误您日常工作。
```

## 模板 D · 给普通员工专用（最轻松）

```
@小李

下午我想拉你进一个新的 AI 助手试用群，5-10 个同事一起。

简单说就是：飞书里多个机器人，能帮你
  - 写邮件 / 客户消息
  - 做海报文案
  - 整理资料
  - 拆任务

试一周，喜欢就留着用，不喜欢就退群。

愿意试试吗？
```

## 发完之后

让员工有 30 分钟"消化时间"。30 分钟后：

1. 飞书新建群 **「DC-Agent · AI 小助手内测群」**
2. 拉所有同意的员工进群
3. 拉巅池-Agent 小助手机器人进群
4. **第一条消息发**：[飞书卡片_欢迎.json](飞书卡片_欢迎.json) 内容（飞书后台 → 消息卡片 → 粘贴 JSON）
5. **第二条消息发**：[员工使用指南.html](员工使用指南.html) 链接或附件
6. **第三条 @ 全员**："各位下午好，按上面的卡片说明，群规请遵守～大家可以挑一个想试的方向开始 @ 小助手了"

然后等大家自然玩。

---

## 提示

- 不要群发同一份私聊（飞书会折叠），逐个发 + 改下称呼最有温度
- 老板优先单独沟通（让他知道这是有控制的测试，不是"突袭")
- 部门负责人重要 —— 他认可了，部门骨干才会真正参与
