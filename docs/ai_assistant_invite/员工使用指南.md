# 巅池-Agent 小助手新手使用指南

面向：中台-策划部 & 客户部  
更新：2026-06-01  

这份指南先给中台同事看。目标很简单：让策划部、客户部先把小助手用在真实工作里，少写重复文字，少整理重复材料，把更多时间留给判断、沟通和推进。

## 先说清楚：它现在能做什么

### 写草稿

客户话术、活动说明、汇报结构、复盘提纲、群通知，都可以先让它出第一版。

### 整理材料

你把文字、聊天记录、或有权限访问的飞书文档内容给它，它可以帮你提炼重点、归类和改成清单。

### 拆工作

项目推进、活动执行、客户跟进，可以让它拆成步骤、时间点、负责人和待确认问题。

### 按部门理解任务

系统里已经有“策划”和“客户部”的常见工作场景，问得越贴近业务，它越容易给出可用结果。

## 不要把它当什么

- 它不是客户本人，不能替你承诺价格、资源、排期和最终方案。
- 它不是公司制度和合同的最终解释人，涉及合同、报价、财务、法务要找负责人确认。
- 它不知道你没发给它的背景。资料越少，它只能越多猜测；资料越具体，结果越接近可直接使用。

## 第一次怎么用

在飞书里私聊小助手，或在群里 @ 它，直接说你想让它帮什么。第一次使用时，如果系统还不认识你，可能会让你选择部门、岗位、填写姓名，并完成一小段入门引导。

```text
@巅池-Agent小助手 帮我把下面这段客户需求整理成：背景、客户想要什么、我们要补充确认什么、下一步怎么推进。
```

如果结果不合适，不用重新开始，继续追问就行：

```text
太长了，改成老板 1 分钟能看完的版本。
再加一个“需要客户确认的问题”清单。
语气不要太硬，适合发给客户看。
```

## 策划部推荐用法

### 1. 做项目进度汇报框架

适合周报、阶段汇报、向上同步。

```text
@巅池-Agent小助手 帮我整理一个项目进度汇报框架。
项目：五菱红标新媒体运营
当前进展：……
遇到的问题：……
下周要推进：……
```

### 2. 做洞察和竞品简报

适合先搭分析结构，再由策划同事补判断。

```text
@巅池-Agent小助手 帮我把这些竞品信息整理成一页简报：
1. 主要打法
2. 值得借鉴的点
3. 对我们项目的启发
4. 需要继续查证的信息
```

### 3. 整理提案结构

适合从零搭目录、梳理逻辑，不适合直接当最终策略。

```text
@巅池-Agent小助手 我们要做一个新媒体传播提案，目标是提升品牌声量和线索转化。
请先帮我搭一个提案目录，并说明每一页应该放什么内容。
```

### 4. 做复盘初稿

适合把零散结果先归整成“发生了什么、好在哪里、问题在哪”。

```text
@巅池-Agent小助手 根据下面的活动记录，帮我整理复盘：
亮点、问题、数据表现、客户反馈、下次优化建议。
```

## 客户部推荐用法

### 1. 写客户触达话术

适合节日问候、回访、邀约、资料补充提醒。

```text
@巅池-Agent小助手 帮我写一段客户回访话术。
对象：老客户
目的：了解近期项目反馈，并自然引出后续合作机会
语气：真诚、简洁、不强推
```

### 2. 整理客户需求

适合把聊天记录变成清楚的需求清单。

```text
@巅池-Agent小助手 帮我把下面客户反馈整理成：
客户明确说了什么、我们理解的需求、还没确认的问题、建议下一步动作。
```

### 3. 做活动/渠道计划初稿

适合先出方向和清单，再由客户部补客户背景和资源限制。

```text
@巅池-Agent小助手 帮我做一个客户渠道推广计划初稿。
品牌/项目：……
目标：……
目标客户：……
已有资源：……
时间：……
```

### 4. 生成跟进清单

适合会后、客户沟通后，把待办和责任人先列出来。

```text
@巅池-Agent小助手 根据下面的沟通记录，帮我整理客户跟进清单：
待办事项、负责人、建议截止时间、需要客户确认的问题。
```

## 好用的提问方法

| 你要告诉它 | 示例 |
|---|---|
| 你是谁 | 我是客户部，准备给客户发一段回访话术。 |
| 你要做什么 | 帮我整理成汇报框架 / 写成客户能看的版本 / 拆成待办清单。 |
| 背景材料 | 客户反馈、项目进度、已有方案、会议记录、文档链接。 |
| 输出格式 | 用表格列出 / 控制在 300 字内 / 分成 4 个部分。 |
| 语气要求 | 正式一点 / 适合发客户 / 老板能快速看懂 / 不要太像 AI。 |

## 可以直接复制的模板

### 模板 1：整理客户需求

```text
@巅池-Agent小助手 我是客户部。请把下面内容整理成客户需求清单：
1. 客户明确提出的需求
2. 我们需要补问的问题
3. 建议下一步动作
4. 可以发给客户确认的简短回复

材料如下：
……
```

### 模板 2：搭策划提案结构

```text
@巅池-Agent小助手 我是策划部。请帮我搭一个提案结构。
项目背景：……
目标：……
客户关心点：……
已有材料：……
请输出：目录、每一页要讲什么、还缺哪些材料。
```

### 模板 3：会议后整理待办

```text
@巅池-Agent小助手 请把下面会议记录整理成：
会议结论、待办事项、负责人、截止时间、风险提醒。
如果记录里没有负责人或截止时间，请标“待确认”。

记录如下：
……
```

### 模板 4：把文字改得更好发

```text
@巅池-Agent小助手 请把下面这段话改成适合发给客户的版本。
要求：真诚、简洁、有礼貌，不夸大承诺。

原文：
……
```

## 使用边界

| 事情 | 能不能让小助手做 |
|---|---|
| 整理客户聊天记录 | 可以，但不要发客户隐私、账号密码、合同敏感条款。 |
| 写客户回复草稿 | 可以，发送前必须由负责人看一遍。 |
| 做策略判断 | 可以让它给思路，但最终判断由策划/客户负责人决定。 |
| 报价、合同、付款、法律表述 | 不要直接采用它的结论，必须人工确认。 |
| 读取你发来的材料 | 可以尝试。读不到或资料不足时，请把关键内容直接粘贴给它。 |

## 反馈方式

这次先让中台核心业务部门试用，最需要你们反馈两类问题：

1. 哪些回答可以直接省时间。
2. 哪些回答不懂业务、说空话、方向跑偏。

反馈时最好带上截图，并补一句：你原本希望它帮你得到什么结果。这样后续才能把系统调得更贴近策划部和客户部的真实工作。

## 一句话记住

小助手适合先帮你“起草、整理、拆解、改写”。涉及客户承诺、策略定稿、价格合同和关键数据时，它只能做辅助，最后一定由人确认。
