# AstrBot 应急回滚手册

> **日期**：2026-05-15
> **适用范围**：周一群测试期间（5/18 - 5/24）以及之后任何"机器人翻车"场景
> **使用频率**：希望永远用不上，但用上时要 30 秒内找到要执行的命令

---

## 🚨 紧急程度分级

| 级别 | 现象 | 响应时间 | 行动 |
|---|---|---|---|
| **P0 危机** | 机器人在群里发了敏感数据 / 违法内容 | **< 1 分钟** | 立即停服（章节 1）+ 删除消息 |
| **P1 严重** | 机器人持续不回 / 大量错答 / 成本暴涨 | < 10 分钟 | 重启或回滚（章节 2-3）|
| **P2 一般** | 偶尔答错 / 体验小 bug | 当天处理 | 记录 + 周末改 |

## 1️⃣ P0 危机 · 立即停服（30 秒）

### Step 1 · 关停 AstrBot

```bash
launchctl bootout gui/$(id -u)/io.astrbot.bot
```

效果：AstrBot 进程立即终止 → 飞书机器人**不再响应任何消息**。

验证：
```bash
launchctl list io.astrbot.bot 2>&1 | grep -q 'Could not find' && echo "✅ 已停" || echo "❌ 还在跑"
```

### Step 2 · 飞书侧删除问题消息

1. 进入飞书测试群
2. 长按机器人发的问题消息 → 撤回（2 分钟内可撤回）
3. 如果超 2 分钟无法撤回 → 在群里发一条说明"刚才那条机器人输出有误请忽略"

### Step 3 · 通知老板 + 群成员

```
@群成员 机器人出现异常已临时下线，正在排查。
今天大家先用人工，明天再恢复。
```

### Step 4 · 删除数据库里的问题对话（重要）

```bash
cd /Users/dianchi/DC-Agent
# 找到该会话的 conversation_id
sqlite3 data/data_v4.db "SELECT id, user_id, created_at FROM conversations ORDER BY created_at DESC LIMIT 10;"

# 删除指定会话（替换 <id>）
sqlite3 data/data_v4.db "DELETE FROM conversations WHERE id = '<id>';"
sqlite3 data/data_v4.db "DELETE FROM platform_message_history WHERE conversation_id = '<id>';"
```

### Step 5 · 复盘记录

写一份事故记录到 `data/watchdog/incidents/p0-<日期>-<事件简称>.md`，下次启用前必须读。

## 2️⃣ P1 严重 · 配置回滚

### 触发场景

- LLM 回答持续不对路（比如全部回英文、全部胡说八道）
- 成本暴涨（每小时超 ¥50）
- AstrBot 启动报错日志一片红

### 操作步骤

```bash

launchctl bootout gui/$(id -u)/io.astrbot.bot

cp data/_backup_20260515_165648/abconf_4b8957bf-4ed1-4e78-a702-aa6ea7ff2fb0.json data/config/
cp data/_backup_20260515_165648/abconf_b45ba85f-e054-48c4-99d6-1ba42e898046.json data/config/

# Step 3 · 重启 AstrBot
```

## 3️⃣ P1 严重 · 单点重启（不回滚）

很多时候只是 lark 长连接断了，**重启即可恢复**，不需要回滚配置。

```bash
# 一键重启（kickstart -k 等同于"先 kill 再 start"）
launchctl kickstart -k gui/$(id -u)/io.astrbot.bot

# 等 30 秒
sleep 30

# 验证
bash scripts-tools/verify_llm_after_restart.sh
```

### Hermes 卡死的话也可以单独重启

```bash
launchctl kickstart -k gui/$(id -u)/ai.hermes.gateway
launchctl kickstart -k gui/$(id -u)/ai.hermes.dashboard
```

## 4️⃣ 成本爆炸应急（暂停 aihubmix key）

如果 LLM 消耗失控（比如某个员工把机器人当 ChatGPT 刷了几千条）：

### 最快办法 · 暂停 key（aihubmix 后台操作）

1. 打开 https://aihubmix.com → API Keys 页面
2. 找到 **dianchi** key
3. 点"禁用" / "暂停"
4. 立即生效 —— 机器人下一次调用就会 401，AstrBot log 报错但不再产生费用

### 恢复 key

1. 找出谁刷的（飞书后台 / AstrBot conversations 表）
2. 私聊提醒
3. 回 aihubmix 后台启用 key

## 5️⃣ 删除特定员工的所有数据（合规请求）

如果某员工要求"删除我跟机器人的所有对话记录"（GDPR 类合规）：

```bash
cd /Users/dianchi/DC-Agent

# 找到该员工的 open_id
sqlite3 data/employees.db "SELECT open_id, display_name FROM employees WHERE display_name LIKE '%张三%';"

# 假设 open_id 是 ou_abc123
OPEN_ID="ou_abc123"

# 删除会话历史
sqlite3 data/data_v4.db "DELETE FROM conversations WHERE user_id LIKE '%$OPEN_ID%';"
sqlite3 data/data_v4.db "DELETE FROM platform_message_history WHERE user_id LIKE '%$OPEN_ID%';"
sqlite3 data/data_v4.db "DELETE FROM platform_sessions WHERE session_id LIKE '%$OPEN_ID%';"

# 删除档案 + 记忆
sqlite3 data/employees.db "DELETE FROM employees WHERE open_id = '$OPEN_ID';"
sqlite3 data/employees.db "DELETE FROM employee_memories WHERE open_id = '$OPEN_ID';"

# 删除偏好
sqlite3 data/data_v4.db "DELETE FROM preferences WHERE scope_id LIKE '%$OPEN_ID%';"

echo "✅ 该员工数据已清除"
```

完成后给员工**书面确认**（飞书消息截图存档）。

## 7️⃣ launchd 服务管理常用命令

```bash
# 列出所有 DC-Agent 相关进程
launchctl list | grep -E 'astrbot|hermes|openclaw|dcagent'

# 启动一个服务
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/<name>.plist

# 停止一个服务
launchctl bootout gui/$(id -u)/<label>

# 重启（kill 后立即重启）
launchctl kickstart -k gui/$(id -u)/<label>

# 看一个服务的最近日志
log show --process python3.12 --last 5m | grep <关键字>
```

DC-Agent 已有的 launchd 服务标签：
- `io.astrbot.bot` —— AstrBot 主进程
- `ai.hermes.gateway` —— Hermes Gateway
- `ai.hermes.dashboard` —— Hermes WebUI
- `com.dcagent.feishu-sync` —— 飞书云盘同步（每小时）
- `com.dcagent.nas-watchdog` —— NAS 看门狗
- `com.dcagent.baidu-nas-sync` —— 百度网盘 → NAS 同步
- `com.dcagent.nas-status-mail` —— NAS 状态邮件

## 8️⃣ 看门狗告警怎么读

`data/watchdog/state.json` 里每个 service 有：

```json
"巅池-Agent小助手_long_connection": {
  "status": "ok" | "fail:reason",
  "since": "2026-05-18T09:00:00Z"
}
```

如果 `status` 不是 `ok`：

| 服务 | 状态 | 含义 |
|---|---|---|
| `astrbot_dashboard` | fail | AstrBot 主进程挂了 → 立即 `launchctl kickstart -k io.astrbot.bot` |
| `hermes_gateway` | fail | Hermes 挂了 → Hermes 深度任务受影响（飞书聊天不影响）|
| `feishu_sync_heartbeat` | fail:stale_*s | 飞书云盘同步 1+ 小时没跑 → 检查 launchd 状态 |
| `system_entries_plugin` | fail | dashboard 顶部"系统入口"挂了 → 不影响业务 |
