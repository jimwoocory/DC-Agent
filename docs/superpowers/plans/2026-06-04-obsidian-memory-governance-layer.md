# Obsidian Memory Governance Layer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn Obsidian from a read-only source mirror into the human governance layer for DC-Agent memory: review, correct, approve, reject, merge, and publish governed memories back into the runtime recall system.

**Architecture:** `data/nas_memory.db` remains the raw/source index and runtime recall source; Obsidian becomes a structured review workspace generated from memory candidates. Human decisions in Obsidian are synced back into a governed memory database/API, then promoted into `nas_memory.db`, AstrBot KB metadata, and Agent recall prompts/tools with provenance and status.

**Tech Stack:** Python 3.12, SQLite, pathlib, pytest, AstrBot dashboard aiohttp routes, existing `nas_sync/dc_memory_indexer.py`, existing `dc_engines/dc_engines/obsidian_review.py`, Obsidian Markdown frontmatter, optional Obsidian Dataview/Tasks conventions.

### Superseded / Should Be Treated as Deprecated

| Old Path or Idea | Why Deprecated | Replacement |
|---|---|---|
| Pure RawRefs-only Obsidian model | It is readable, but it cannot safely control what the agent believes. | Governance notes with import/promote/audit. |
| JSONL-only review confirmations | Append-only is useful, but insufficient for canonical current state and idempotent promotion. | `governed_memory.db` plus append-only `memory_decisions` and `memory_audit_log`. |
| AGY candidate output as review queue | It is batch-analysis output, not stable product infrastructure. | Deterministic exporter from source DB to Obsidian governance notes. |
| Directly editing `nas_memory.db` by hand | No audit trail, easy to corrupt source index. | Obsidian decision -> importer -> governed store -> promoter. |
| Treating AstrBot KB mirror as the main memory system | Current project notes say AstrBot mirroring is optional for DC-Agent recall. | Keep NAS memory/governed memory as primary; mirror only when needed for compatibility. |

### Not Needed for First Production Version

| Candidate Feature | Reason to Skip Initially | Revisit Trigger |
|---|---|---|
| Custom native Obsidian plugin | Markdown frontmatter plus CLI/dashboard is enough for v1. | Add plugin only if reviewers need buttons, validation UI, or one-click approve inside Obsidian. |
| Real-time file watcher for Obsidian edits | Scheduled/manual import is safer and easier to audit. | Add watcher after import idempotency is proven. |
| Vector embeddings for governed memory | Raw NAS/KG retrieval already exists; governance first needs correctness and status filtering. | Add when approved memory volume is large and lexical search is insufficient. |
| Multi-user RBAC in Obsidian | File-based review can start with trusted internal reviewers. | Add when multiple departments review sensitive/conflicting facts. |
| Automatic deletion/forgetting | Risky for source-of-record documents. | Add only after retention policy is approved. |
| Full AstrBot KB write-back | Current DC-Agent recall does not require it. | Add only for assistants that cannot query NAS/governed memory directly. |
| Bulk proactive review pings | Existing strategy explicitly avoids batch interruption. | Add only as opt-in review campaign workflow. |

### New Governance Loop

```text
NAS / Feishu / chat / manual facts
        |
        v
raw memory candidates in SQLite
        |
        v
Obsidian Review Notes with stable frontmatter IDs
        |
        v
human edit / approve / reject / merge / mark sensitive
        |
        v
sync back into governed_memory.db
        |
        v
promotion worker updates recall indexes + overrides + audit log
        |
        v
Agent answers with governed facts, citations, and confidence
```

### Core Product Rules

1. Obsidian is not the source of raw truth. It is the source of human governance decisions.
2. Every editable note has a stable `memory_id`, `source_path`, `source_hash`, `review_status`, and `governance_version`.
3. Runtime recall uses only `approved` or explicitly `auto_approved` memory by default.
4. `need_review`, `rejected`, and `sensitive_blocked` memories can be surfaced to operators, but not used as confident facts in user answers.
5. All promotions are append-only audited.
6. Obsidian edits must be reversible and idempotent.

## Data Model

### SQLite: `data/governed_memory.db`

`governed_memories`

- `memory_id TEXT PRIMARY KEY`
- `source_system TEXT NOT NULL` values: `nas`, `astrbot_kb`, `conversation`, `manual`, `harness`
- `source_id TEXT NOT NULL`
- `source_path TEXT DEFAULT ''`
- `source_hash TEXT NOT NULL`
- `title TEXT NOT NULL`
- `summary TEXT NOT NULL`
- `canonical_text TEXT NOT NULL`
- `memory_kind TEXT NOT NULL` values: `document`, `fact`, `project`, `person`, `process`, `preference`, `decision`
- `review_status TEXT NOT NULL` values: `need_review`, `approved`, `rejected`, `merged`, `sensitive_blocked`, `stale`
- `confidence REAL NOT NULL`
- `sensitivity TEXT NOT NULL` values: `public`, `internal`, `confidential`, `secret`
- `owner TEXT DEFAULT ''`
- `project_id TEXT DEFAULT ''`
- `tags_json TEXT NOT NULL`
- `links_json TEXT NOT NULL`
- `obsidian_note_path TEXT DEFAULT ''`
- `governance_version INTEGER NOT NULL`
- `created_at TEXT NOT NULL`
- `updated_at TEXT NOT NULL`
- `approved_at TEXT DEFAULT ''`
- `approved_by TEXT DEFAULT ''`

`memory_decisions`

- `decision_id TEXT PRIMARY KEY`
- `memory_id TEXT NOT NULL`
- `decision TEXT NOT NULL`
- `reviewer TEXT NOT NULL`
- `reason TEXT NOT NULL`
- `before_json TEXT NOT NULL`
- `after_json TEXT NOT NULL`
- `created_at TEXT NOT NULL`

`memory_audit_log`

- `audit_id TEXT PRIMARY KEY`
- `memory_id TEXT NOT NULL`
- `action TEXT NOT NULL`
- `actor TEXT NOT NULL`
- `payload_json TEXT NOT NULL`
- `created_at TEXT NOT NULL`

def test_store_upserts_memory_and_reads_it_back(tmp_path: Path):
    store = MemoryGovernanceStore(tmp_path / "governed_memory.db")
    store.initialize()

    store.upsert_memory(sample_memory())

    assert loaded is not None
    assert loaded.memory_id == "mem_test_001"
    assert loaded.review_status == "need_review"
    assert loaded.tags == ["customer-a", "delivery"]


def test_decision_and_audit_records_are_append_only(tmp_path: Path):
    store = MemoryGovernanceStore(tmp_path / "governed_memory.db")
    store.initialize()
    store.upsert_memory(sample_memory())

    decision = ReviewDecision(
        decision_id="dec_001",
        memory_id="mem_test_001",
        decision="approved",
        reviewer="dianchi",
        reason="Verified source document.",
        before={"review_status": "need_review"},
        after={"review_status": "approved"},
        created_at="2026-06-04T00:01:00Z",
    )
    store.record_decision(decision)
    store.append_audit("mem_test_001", "import_decision", "dianchi", {"decision": "approved"})

    assert len(store.list_decisions("mem_test_001")) == 1
    assert len(store.list_audit("mem_test_001")) == 1

### Phase 4: Scheduled Governance

## Operational Policy

- Human reviewer must set `reviewer` for `approved`, `rejected`, `merged`, or `sensitive_blocked`.
- `approved` requires non-empty `Canonical Memory`.
- `merged` requires non-empty `merged_into`.
- `sensitive_blocked` must never be promoted into user-facing recall.
- `secret` memories can be stored for audit but are excluded from default dashboard and recall results.
- Every sync run writes JSON status to `data/watchdog/obsidian_memory_governance_state.json`.
