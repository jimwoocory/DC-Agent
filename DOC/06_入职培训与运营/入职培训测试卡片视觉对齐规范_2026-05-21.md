# 入职 / 培训 / 培训测试卡片视觉对齐规范

目标：让 Claude Code 侧的入职登记、培训教程、培训测试卡，向 DC-Agent 当前卡片风格看齐。本文只规定视觉、排版、交互样式，不调整培训内容和测试题内容。

参考实现：

- `dc_engines/dc_engines/feishu_card_streamer/templates.py`
- 核心函数：`build_onboarding_*`、`build_tutorial_lesson_card`、`build_quiz_*`
- 公共按钮函数：`_button_row`

## 1. 总体风格

所有入职 / 培训 / 测试卡统一采用“业务卡片”风格：

- 标题简洁，不在 header 里堆 emoji。
- Header 颜色：
  - 入职登记：`green`
  - 培训清单：`green`
  - 培训教程：`green`
  - 培训测试题：`violet`
  - 答对 / 通过：`green`
  - 答错 / 未通过：`orange`
- 正文第一屏必须先给状态和结构，不要直接进入一大段文字。
- 同一类卡片必须保持字段顺序一致。

## 2. 字体层级

不要滥用 `heading_1`。除非是最终结果类大标题，否则不要用。

推荐：

- 主标题正文：`heading_2`
- 小节标题：`heading_3`
- 正文：`normal`
- 状态、提示、倒计时、说明：`notation`

示例：

```text
测试同学，资料齐了              heading_2
● 入职登记已完成                notation
教程：5 节短教程                 normal
测试：6 题                       normal
规则：全对通过                   normal
```

避免：

```text
🎉 超大标题
正文又很小
按钮再散开
```

## 3. 颜色规范

正文里需要明确区分状态，不要只靠 header 颜色。

使用：

- 成功 / 已完成 / 已通过：`<font color='green'>...</font>`
- 需要确认 / 待作答 / 未通过 / 风险：`<font color='red'>...</font>`
- 辅助说明：`<font color='grey'>...</font>`
- 测试进行中：可用 `violet`

示例：

```md
**状态**：<font color='green'>已通过</font>
**状态**：<font color='red'>未通过</font>
**提示**：<font color='grey'>测试 6 题 · 全对通过</font>
```

## 4. 字段对齐

所有元信息字段用对齐字段块，不要写成散文。

推荐字段顺序：

入职登记：

```md
**步骤**　：1 / 3
**状态**　：<font color='green'>入职登记中</font>
**下一步**：选择所在部门
```

培训清单：

```md
**教程**：5 节短教程
**测试**：6 题
**规则**：全对通过
```

教程详情：

```md
**章节**：第 2 / 5 节
**状态**：<font color='green'>学习中</font>
**时长**：约 1-2 分钟
```

测试题：

```md
**进度**：1 / 6
**题型**：单选题
**状态**：<font color='red'>待作答</font>
```

测试结果：

```md
**成绩**：6 / 6
**状态**：<font color='green'>已通过</font>
**下一步**：加入内测群
```

## 5. 按钮规范

这是目前最重要的对齐点。

所有按钮必须视觉上稳定、居中、等宽，不允许文字撑宽导致按钮大小不一。

公共规则：

- 按钮默认 `width: fill`。
- 单按钮：用左右 spacer 居中，不要贴左。
- 双按钮：用左右 spacer 收住版心，不要一个左一个右。
- 三个及以上按钮：按等宽列铺开。
- 测试题 A/B/C/D：按钮只显示 `A` / `B` / `C` / `D`，选项正文放在按钮上方。

错误示例：

```text
[继续学下一节]                         [开始测试]
```

正确示例：

```text
        [继续学下一节]   [开始测试]
```

测试题正确结构：

```md
**选项**
- **A**：不用特别说，默认就是方版
- **B**：直接说「画一张方版的...」
- **C**：只能自己后期裁
- **D**：默认竖版，要说「横版」才会横

[A] [B] [C] [D]
```

不要把长选项塞进按钮里。

## 6. 教程卡规范

教程卡 header 已经是教程标题，正文不要重复一个大标题。

结构：

```text
Header: 生图 · 两套用法

● 培训教程
章节：第 2 / 5 节
状态：学习中
时长：约 1-2 分钟

教程正文...

[继续学下一节] [开始测试]
60 秒不点的话，自动继续下一节。
```

教程正文里的 markdown 表格必须转成居中表格行，不能直接让飞书原生 markdown 表格乱渲染。

## 7. 入职登记三步

### 第 1 步：选择部门

必须包含：

- 欢迎语
- 状态行：等待选择部门
- 对齐字段：步骤 / 状态 / 下一步
- 部门按钮等宽分组
- 灰色提示：点错可 `/重新入职`

### 第 2 步：选择角色

必须包含：

- 已选部门
- 状态行：部门已记录
- 对齐字段：步骤 / 部门 / 下一步
- 角色按钮等宽

### 第 3 步：输入姓名

必须包含：

- 已选角色
- 状态行：角色已记录
- 对齐字段：步骤 / 角色 / 下一步
- 输入姓名说明
- 灰色示例

## 8. 培训测试卡

题目卡：

- Header：`测试 · 第 N / Total 题`
- 状态行：培训测试进行中
- 对齐字段：进度 / 题型 / 状态
- 题干用 `heading_2`
- 选项正文正常展示
- 按钮只显示 A/B/C/D

答题反馈：

- 答对：green header，状态 `<font color='green'>正确</font>`
- 答错：orange header，状态 `<font color='red'>需复习</font>`
- 必须有对齐字段：题目 / 结果 / 下一题
- 解析单独成段，格式：`**解析**：...`

结果卡：

- 通过：green header
- 未通过：orange header
- 首屏必须显示：成绩 / 状态 / 下一步
- 按钮居中且等宽

## 9. 验收清单

改完后至少验证：

- [ ] 单按钮不贴左，视觉居中。
- [ ] 双按钮不贴左右边，左右留白一致。
- [ ] A/B/C/D 四个按钮等宽。
- [ ] 长选项没有进入按钮内部。
- [ ] 入职三步都有 `步骤 / 状态 / 下一步` 对齐字段。
- [ ] 培训教程没有重复大标题。
- [ ] 教程表格居中，字号稳定。
- [ ] 测试反馈和结果有红绿状态字。
- [ ] 正文不再出现标题过大、正文过小的断层。
- [ ] 飞书真机卡片截图看起来像同一套系统。

## 10. 不要改的部分

本轮不要改：

- 培训内容文案
- 测试题题干
- 测试题答案
- 部门题逻辑
- 入职状态流转逻辑

内容准确性另开一轮，让 Claude Code 专门复核。
