# JW-Bot 系统架构总览

最后更新：2026-05-29

## 整体架构

```
员工（QQ / 飞书 / WebChat）
         ↓ 消息
    AstrBot 核心
    ┌─────────────────────────────────────────┐
    │  Pipeline                                │
    │  RouterStage → ProcessStage → RespondStage│
    │       ↓              ↓                   │
    │  意图分类       Star Handlers             │
    │  任务/技能/对话  LLM Agent               │
    └──────────┬──────────────────────────────┘
               │
     ┌─────────┴──────────┐
     │                    │
  Harness 系统        hermes_bridge
  任务全生命周期        ↕ Webhook
  记忆提升             Hermes Agent
```

## 消息处理 Pipeline

```
1. RouterStage     意图分类（rules + LLM fallback）
                   → task: 创建 Harness 任务
                   → skill: 激活 Star handler
                   → conversation: 透传

2. ProcessStage    Star handlers 处理（onboarding / builtin_commands / plugins）
                   → 如无 handler 结果，走 LLM Agent

3. InternalAgentSubStage   LLM 生成响应
                           → _maybe_complete_harness_task（自动闭环）

4. RespondStage    发送回平台
```

## 关键数据存储

| 路径 | 存储内容 | Git 管理策略 |
|------|----------|--------------|
| `data/plugins/` | DC-Agent 自定义插件代码、插件页面与 schema | 按插件源码管理；禁止提交 `__pycache__`、`.pyc` 和备份文件 |
| `data/config/hermes_bridge_config.json` | Hermes Bridge 非密钥配置 | 可管理；生产密钥不得写入 |
| `data/config/openclaw_on_demand_config.json` | OpenClaw 按需入口配置 | 可管理；生产密钥不得写入 |
| `data/config/system_entries_config.json` | Dashboard 快捷入口配置 | 可管理；生产密钥不得写入 |
| `data/feishu_whitelist.example.yaml` | 飞书白名单示例 | 可管理；真实白名单不提交 |
| `data/data_v4.db` | 会话历史、Persona、session 配置（sp preferences） | 运行时数据，不提交 |
| `data/lossless_context.db` | 无损压缩上下文（lossless_items / heads） | 运行时数据，不提交 |
| `data/harness.db` | Harness 任务记录 | 运行时数据，不提交 |
| `data/harness_memory.db` | LLM 响应提升的记忆 | 运行时数据，不提交 |
| `data/knowledge_base/` | 知识库索引、上传文件与 chunk 数据 | 运行时数据，不提交 |
| `data/hermes_sessions.db` | hermes_bridge 的 platform_user → session_key 映射 | 运行时数据，不提交 |
| `data/temp/`、`data/output/`、`logs/`、`output/` | 临时文件、生成物、日志 | 运行时产物，不提交 |

## 人格（Persona）配置

| Persona ID | 适用角色 | 备注 |
|------------|----------|------|
| `Biz_Assistant_Claw` | 推广/运营/内容 | 默认推荐 |
| `Enterprise_Ops_Kernel` | 项目管理/跟进 | |
| `DevOps_Console` | 技术/运维 | |

人格存储在 `data/data_v4.db` 的 `personas` 表。  
Session 级别绑定通过 `sp`（preferences 表）的 `session_service_config.persona_id` 实现。

## 网络拓扑

```
Mac mini（AstrBot）
    ↓ 默认网关
路由器 192.168.8.1（Surge TUN）
    ↓
互联网
    ├─ api.sgroup.qq.com（QQ 官方 Bot API）→ DIRECT 规则
    ├─ open.feishu.cn（飞书 API）→ 需稳定路由
    └─ aihubmix / OpenAI / 其他 LLM API → 代理
```

**注意**：QQ 和飞书 API 域名建议在路由器 Surge 配置 DIRECT 规则，避免代理抖动导致连接 reset。
