## ~~OpenClaw on-demand 看门狗~~ ✅ 完成（2026-05-14 15:11）

**背景**：OpenClaw Control Center 是"可视化插件"性质——偶尔才看，不该常驻吃 200MB 内存。但又希望"点 dashboard 按钮就能用"，不想 terminal 手动启。

**做法**：新增 plugin [`data/plugins/openclaw_on_demand/`](data/plugins/openclaw_on_demand/)
- `main.py` 起 aiohttp `:9120`，暴露 `/kick` `/status` `/stop`
- `/kick` 检查 4312 listen → 没起就 `subprocess.Popen(["npm","run","dev:ui"], cwd=Openclaw, start_new_session=True)` → 等 ready → 302 跳 4312
- background asyncio task 每 60s 检查 last_kick_at，超 1800s 自动 `killpg` 整个进程组
- AstrBot 重启时**接管**已运行的 OpenClaw（读 PID 文件验活）
- `_conf_schema.json` 7 个字段（cwd / 端口 / idle_timeout / npm_cmd / log_dir 都能在 dashboard 改）

**dashboard 按钮**：`localhost:4312` → `localhost:9120/kick`（[FullLayout-kick130d127f.js](data/dist/assets/FullLayout-kick130d127f.js)）

**验证**：
- `GET /status` → JSON 返回正常
- `GET /kick` → HTTP 302 → `http://127.0.0.1:4312/`
- Follow → HTTP 200 + title="OpenClaw Control Center"
- 重启 AstrBot → plugin 自动接管现有 OpenClaw PID

**故意没做的事**：
- ❌ 不加 launchd 让 OpenClaw 开机自启（违背"可视化插件"语义）
- ❌ idle kill 时不发通知（属背景活动，无需打扰）

## ~~AstrBot ↔ Hermes HMAC secret 对齐 + plugin schema~~ ✅ 完成（2026-05-14 11:20）

**问题**：升级完 Hermes 后 AstrBot 派任务给 Hermes 全部被 `401 Invalid signature` 拒——
- Hermes 那边 secret: `${HERMES_BRIDGE_SECRET}`（hermes-config/config.yaml）
- AstrBot 那边 secret: 默认值 `astrbot-secret-key`（plugin hardcoded default）
- 直接往 cmd_config.json 加 `hermes_bridge` 段会被 AstrBot v4.24+ 的 schema 整合机制清除：`Config key removed: hermes_bridge`

**做法**：给 hermes_bridge plugin 加 `_conf_schema.json` + 改 `__init__(self, context, config=None)` 接 plugin-scoped config

**文件**：
- 新增 [`data/plugins/hermes_bridge/_conf_schema.json`](data/plugins/hermes_bridge/_conf_schema.json) — 9 个字段（secret/url/port/platforms/memory_inject/...）
- 改 [`data/plugins/hermes_bridge/hermes_bridge.py:148`](data/plugins/hermes_bridge/hermes_bridge.py:148) — 优先 plugin-scoped config，回退 context.get_config()
- 自动生成 `data/config/hermes_bridge_config.json` — 可在 dashboard 编辑

**验证**：
- AstrBot → Hermes: `POST /webhooks/astrbot_task` → **HTTP 202** + `inbound message: platform=webhook user=astrbot_task ...`
- Hermes → AstrBot: `POST /task_result` + `POST /hermes_response` → **HTTP 200**
- AstrBot dashboard → 插件管理 → hermes_bridge → 现在可以可视化改 secret / URL / 白名单等 9 项配置
