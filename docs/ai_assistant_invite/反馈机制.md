# 反馈机制 · 整周用

> **每日反馈话术 + 周末复盘模板**
> 让员工反馈成本最低，让你收集成本也最低

---

## 一、每日反馈 · 下班前 1 分钟版

每天下午 18:00 左右在群里 @ 全员：

```
@全体成员 今日 1 分钟反馈～

1. 今天 @ 过小助手几次？大概问了什么？
2. 印象最深的一次回答（好的 / 差的都行）？
3. 希望明天小助手还能帮你做什么？

简单回 3 句话就行，不用形式。
```

---

### 为什么这样设计

| 设计点 | 原因 |
|---|---|
| 3 个开放式问题 | 比"是 / 否"更有信息量 |
| 不用表单 | 飞书表单要切窗口、要打字 → 员工嫌烦不填 |
| "简单回 3 句话" | 降低心理门槛（要求"完整回答"反而没人填）|
| 下班前发 | 员工状态放松、记忆新鲜 |
| @ 全员 | 群提示音 + 同伴效应（看别人回了自己也回）|

## 二、整周复盘 · 5/24 周日晚

把这一周收集到的反馈整理成一页给老板，结构：

### 📊 数据看板（量化）

| 指标 | 数 |
|---|---|
| 累计 @ 小助手次数 | XXX |
| 累计反馈回复数 | YYY |
| 印象最深的"好" | 引用 3 条员工原话 |
| 印象最深的"差" | 引用 3 条员工原话 |
| 员工想要的新功能 | 列出 Top 5 |

### 💬 质化总结（员工原声）

| 部门 | 员工 | 一句话评价 |
|---|---|---|
| 市场部 | 罗鸣 | "..." |
| 品宣部 | 黄丽清 | "..." |
| 策略部 | 谭媛尹 | "..." |
| ... | ... | ... |

### 🔧 改进项（你 PM 视角）

按"现在做 / 本月做 / 下次再说"分类：

- **🔴 立刻做**（影响体验的 bug、错答）：
  - [ ] 例：小助手不会写飞书表格里的公式 → 加 KB 学习
  - [ ] 例：小助手聊天时偶尔回英文 → 改 prompt
- **🟡 本月做**（员工提到 2 次以上的新功能）：
  - [ ] 例：员工想"上传图片让 AI 看"→ 接多模态
- **⚪️ 下次再说**（只 1 个员工提，或者复杂）：
  - [ ] 例：员工想"AI 帮我接客户电话"→ 暂时不做

### 🎯 老板的决策点

写 2-3 个老板需要拍板的问题：

- 要不要 5/25 开始扩到全员？
- 要不要给 X 部门加专用 KB？
- 要不要买 X 增值功能？

## 三、突发反馈 · 立即处理

**触发条件**：员工**主动在群里**说"机器人答错了"/"这个不对"。

**你的标准动作**：
1. 立刻 @ 该员工 + 道歉 ("感谢反馈，让我看看")
2. 截图保存
3. 群里**不公开追责机器人**（避免引起其他员工的"AI 不靠谱"印象）
4. 私聊该员工细聊（"具体哪里不对？你期望的回答是什么？"）
5. 进入"立刻做"清单

---

## 四、不期望什么

| 不期望 | 原因 |
|---|---|
| 详细书面反馈 | 员工嫌烦 → 反馈率会归零 |
| 100% 反馈率 | 50% 已经足够，能反馈的都是"积极参与者" |
| 全员一致好评 | 真实反馈一定包含负面，**没有差评反而要警惕** |
| 一周搞定所有改进 | 内测目的是"发现问题"不是"修复所有问题"|

---

## 五、关键心态

> **测试期不是验收期**。员工说不好用，不是机器人的耻辱，是**信息**。
>
> 我们花一周搜集真实数据，比让机器人"表演 100 次完美回答"更值钱。
